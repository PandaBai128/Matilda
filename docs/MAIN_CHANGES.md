## 1. 产品身份与个人缘起

- App 显示名改为“壮壮”，英文产品名和可执行文件名使用 `Matilda`；
- 菜单栏、控制面板、语音试听、ASR 关键词和错误提示统一使用壮壮身份；
- 原来的蓝色三角形替换为根据壮壮形象制作的透明头像；
- 应用图标和菜单栏图标同步更换，不再使用旧 Clicky 黑框图标；
- README 明确说明本项目源自 Clicky，但现在是围绕个人陪伴与日常屏幕问答形成的定制版本。

## 2. 跟随鼠标与状态动画

- 壮壮会在鼠标旁保持可配置距离，并提供快、中、慢三档跟随感；
- 鼠标移动时使用短生命周期的 60fps 平滑跟随，停止约 200 毫秒后立即停止更新；
- 空闲状态不再维持永久动画时钟，只在需要眨眼时短暂唤醒；
- 倾听、思考、回答和指向使用不同的按需动画频率；
- 支持自然眨眼、倾听波纹、思考倾斜和到达目标后的两次张嘴“汪，汪汪”；
- 默认鼠标静止 10 秒后自动隐藏，移动鼠标或开始语音交互时重新出现；
- 多显示器共用一个鼠标跟踪源，只在鼠标所在屏幕渲染头像。

## 3. 外观设置

新增独立的 Appearance 窗口，可以实时预览并调整：

- 头像大小；
- 与鼠标的距离；
- 跟随速度；
- 泛光颜色和强度；
- 自动隐藏时间；
- 指向圈大小与颜色；
- 是否显示目标中心点；
- “汪，汪汪”标签的背景颜色。

目标圈的中心始终保留模型返回的真实坐标，即使靠近屏幕边缘也不会为了避免裁切而移动中心。头像和文字标签可以进行边缘约束，但不修改目标点。

## 4. 连续对话和指向稳定性

- “点一下”“点击一下”现在会被识别为视觉指向请求；
- 明确说出一个新目标时，不再把上一轮目标描述和旧坐标带给模型；
- “怎么关掉它”“刚才那个在哪”这类依赖上下文的短追问仍保留上一轮对话；
- 普通知识、写作、抽象问题和地理问题继续保持不指向；
- 当前仍采用一次 MiniMax 截图视觉请求，不使用二次裁剪定位或系统 Accessibility 控件树；
- 坐标是视觉模型根据截图作出的估计，目前适合作为辅助指引，不作为自动点击依据。

## 5. 连续语音与界面响应

- 流式 MP3 的解析、Audio Queue 生命周期和取消清理由专用串行队列处理；
- 用户开始下一轮说话时，可以取消上一轮 TTS，不等待缓慢的底层音频清理；
- 旧的音色试听和旧的流式响应不能在取消后重新播放；
- 网络或服务失败使用中性、准确的系统语音提示，不再误报“额度耗尽”；
- Release 版本不创建 `/tmp/clicky-debug.log`，也不记录转写、回答正文和屏幕衍生文本。

## 6. 构建、安装与权限

- 唯一建议安装位置为 `~/Applications/Matilda.app`；
- 每次通过 Xcode UI 测试后，需要再执行一次普通 Build，再替换安装版；
- 清理 DerivedData 和临时目录中的 `Clicky.app` / `Matilda.app` 副本，避免 Spotlight、Launch Services 和系统权限列表出现多个同名应用；
- 禁止通过终端运行 `xcodebuild`，避免签名和 TCC 权限实例变化。

## 7. 测试与当前边界

当前 Swift 单元测试覆盖：

- POINT 标签解析和坐标映射；
- 屏幕边角目标中心；
- 指向请求的正反例；
- 连续目标上下文；
- 鼠标跟随、自动隐藏和动画频率策略；
- TTS 预览取消、流式会话隔离和非阻塞清理；
- 权限路由和失败提示；
- 敏感正文不得写入日志。

Worker 测试覆盖 MiniMax TTS SSE 分块、上游取消和背压处理。

仍未实现：

- 自动点击、输入或执行脚本；
- 内置互联网搜索；
- 系统原生控件级的百分之百精准定位；
- 不依赖本地代理或 Cloudflare Worker 的直接 API 调用。
